# v0.1.0 Acceptance Checklist — Dogfood & Release (PLAN.md Step 10)

**How to use this document.** Work through the parts in order, checking off each box as you personally verify it. When a box can't be checked, triage it: blockers (rule below) must be fixed before the tag; everything else becomes a post-0.1 backlog issue — note the issue number next to the box. When every box is checked or dispositioned, commit this file under `docs/releases/` as the acceptance record for v0.1.0.

**Triage rule (PLAN.md 10.4).** A finding blocks the tag only if it is: data loss/corruption, silently wrong analysis or report numbers, a security regression against `docs/THREAT_MODEL.md`, or a broken core flow (onboarding, logging a day, Oura sync, dashboard). Everything else → backlog issue.

**Prerequisite.** Step 9 (threat model) is complete and the PR gate is lifted — fixes for anything found here flow through normal PRs.

This checklist was authored 2026-07-06 against the actual code (two full inventories: frontend UI surface and backend API surface). It describes what the app really does today, including known quirks. Anything the app does that *surprises* you beyond what's written here is a finding — log it.
